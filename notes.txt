Safer Alternatives
Option A — Use Only Secure HttpOnly Cookies

Store JWT or session token in HttpOnly, Secure, SameSite cookie.

Server reads cookie directly — no need for localStorage.

SSR-friendly, XSS-resistant, works for all roles.

Option B — Hybrid but Safe

Store access token in memory (not localStorage).

Store refresh token in HttpOnly cookie.

For protected routes, server validates access token, and can issue a new one using refresh token automatically.

This achieves:

XSS resistance (token never exposed to JS)

Automatic server validation

No need for localStorage at all

const handleLogout = async () => {
    setIsAuthenticated(false);
    sessionStorage.removeItem("userType"); // Clear userType from storage
    router.push("/userData/LoginUser"); // Redirect to LoginUser

    try {
      const response = await fetch("../../api/auth/logout", {
        method: "GET",
      });
      const data = await response.json();
      if (!data.success) {
        console.error("Error logging out:", data.message);
      }
    } catch (error) {
      console.error("Error:", error);
    }
  };

<div className="text-right mt-8 mr-8">
      <button className="text-white bg-red-700 hover:bg-red-800 focus:ring-4 
                     focus:ring-blue-300 font-medium rounded-lg text-sm px-5 
                     py-2.5 me-2 mb-2 dark:bg-red-600 dark:hover:bg-red-700 
                     focus:outline-none dark:focus:ring-red-800"
